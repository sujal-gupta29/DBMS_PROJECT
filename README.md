# EstateHub — Real Estate DBMS Project

A complete, production-ready Real Estate Management System built with **FastAPI + MySQL 8 + React 18**.

---

## 🏗️ Tech Stack

| Layer      | Technology                              |
|------------|-----------------------------------------|
| Backend    | Python 3.9+, FastAPI, PyMySQL           |
| Database   | MySQL 8.0                               |
| Frontend   | React 18, Recharts, Lucide-React        |
| Auth       | JWT (python-jose), bcrypt (passlib)     |

---

## 🚀 Quick Start

### Prerequisites
- Python 3.9+
- Node.js 16+
- MySQL 8.0 (running)
- pip, npm

### Linux / macOS — One Command
```bash
bash setup.sh      # Asks for MySQL credentials, sets everything up
bash start.sh      # Starts backend (port 8000) + frontend (port 3000)
```

### Windows — One Command
```bat
setup.bat          # Asks for MySQL credentials, sets everything up
start.bat          # Opens backend + frontend in separate terminal windows
```

### Open in browser
```
http://localhost:3000
```

## 🔐 Role Capabilities

### Admin
- 8-stat KPI dashboard
- Analytics: revenue by month/city, top agents, rent vs sale, locality activity, property type distribution
- Manage Managers: add / deactivate / **reactivate**
- Manage Agents: add (assign to manager) / deactivate / **reactivate**
- View all Clients, Listings, Transactions
- **SQL Console** — live SELECT/WITH/SHOW query runner with result table
- **Academic Queries (a–f)** — dedicated panel with configurable parameters

### Manager
- Team dashboard with agent comparison bar/line charts
- View agent performance with monthly revenue and conversion rate table
- Appointments filtered by all / pending / done
- Team transactions and all listings

### Agent
- Personal performance dashboard with conversion rate stat
- Appointments view: upcoming / pending / done / all
- **Close Deal modal**: record SALE price or full RENT agreement (amount + deposit + end date)
- Auto-updates listing status to SOLD or RENTED on close

### Client
- Browse all active listings with **7 simultaneous filters**: city, type, property type, min bedrooms, min price, max price, locality
- Property detail modal: amenities, full specs, price banner, book button
- Book appointment: choose agent, pick datetime
- My Properties: add properties, create SALE / RENT / BOTH listings
- Appointments history and full transaction history

## 🗄️ Database

### Tables (exact original schema)
MANAGER · AGENT · CLIENT · LOCALITY · PROPERTY · ADDRESS · FEATURES · LISTING · APPOINTMENT · TRANSACTION · RENT_AGREEMENT

### Added
- `AUTH_USERS` — application-level auth (email + bcrypt hash + role)

### Stored Procedures
| Procedure | Description |
|-----------|-------------|
| `sp_add_manager` | Insert manager with UUID |
| `sp_add_agent` | Insert agent (validates active manager) |
| `sp_remove_manager` | Deactivate (blocks if has active agents) |
| `sp_remove_agent` | Deactivate (blocks if has upcoming appointments) |
| `sp_reassign_agent` | Move agent to new manager (validates both) |
| `sp_reschedule_appointment` | Reassign appointment agent (validates future + active) |

### Convenience Views
| View | Description |
|------|-------------|
| `v_listing_full` | Listing joined with property, address, locality, features, owner |
| `v_transaction_full` | Transaction with buyer, seller, agent, manager, rent agreement |
| `v_agent_stats` | Per-agent totals: appointments, deals, revenue, avg sale price |

## 🔧 Manual Setup (if scripts fail)

```bash
# 1. Load database
mysql -u root -p < scripts/setup_db.sql

# 2. Backend
cd backend
cp .env.example .env          # then edit .env with your DB password
python -m venv venv
source venv/bin/activate       # Windows: venv\Scripts\activate
pip install -r requirements.txt
uvicorn main:app --reload --port 8000

# 3. Frontend (new terminal)
cd frontend
npm install --legacy-peer-deps
npm start
```
